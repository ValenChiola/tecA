    VENTAJAS del modelo cliente-servidor

  1. Adminisitracion centralizada
    • Qué significa: Todo el control del sistema se encuentra en el servidor: configuración, acceso, seguridad, etc.
    • Es como una biblioteca centralizada donde todos los libros están organizados y administrados por un bibliotecario. Si necesitás un libro, lo pedís y el bibliotecario lo busca por vos. No hace falta que cada persona sepa dónde está cada cosa.
  2. Acceso controlado globalmente
    • Qué significa: Se pueden definir permisos y políticas de acceso en un solo lugar (el servidor).
    • Analogía: Como una discoteca con un portero en la entrada. Solo deja pasar a quienes están en la lista, y puede cambiar la lista sin tener que hablar con cada invitado.
  3. Estabilidad controlada
    • Qué significa: Es más fácil escalar los recursos del servidor (más RAM, mejor CPU, etc.) que gestionar múltiples clientes independientes.
    • Analogía: Imaginá un restaurante con cocina central. Si hay más clientes, contratás más cocineros o ampliás la cocina, pero no necesitás que cada cliente cocine su comida.
  4. Mantenimiento sencillo
    • Qué significa: Actualizar o reparar el sistema se hace en el servidor, no en cada cliente.
    • Analogía: Como cambiar una bombita en una lámpara central que ilumina todo un salón. No necesitás entrar a cada habitación a cambiar una bombita diferente.
  5. Disponibilidad compartida de recursos
    • Qué significa: Varios clientes pueden usar los mismos archivos, bases de datos, o aplicaciones.
    • Analogía: Como un Netflix corporativo: muchos usuarios ven películas distintas desde el mismo servidor sin tener una copia local.

    DESVENTAJAS del modelo cliente-servidor

  1. Punto unico de falla (caida del servidor)
    • Qué significa: Si el servidor deja de funcionar, todos los clientes quedan sin servicio.
    • Analogía: Es como si se cortara la luz en un shopping: aunque las tiendas estén en buen estado, no pueden operar sin energía.
  2. Costo y complejidad del servidor
    • Qué significa: El servidor debe tener mucha capacidad, buena seguridad, y un mantenimiento constante.
    • Analogía: Es como tener una central eléctrica: necesita inversiones, técnicos, repuestos… pero alimenta muchas casas.
  3. Dependencia de la red
    • Qué significa: La comunicación entre cliente y servidor necesita una red estable y rápida. Si hay cortes o lentitud, afecta la experiencia.
    • Analogía: Como pedir comida por delivery: si el repartidor se retrasa o se rompe la moto, no comés.
  4. Escalabilidad limitada horizontalmente
    • Qué significa: Si hay muchos clientes, el servidor puede saturarse si no está bien diseñado.
    • Analogía: Como una sola ventanilla de atención al público: funciona bien con pocas personas, pero si llega una multitud se genera una cola enorme.
